# Apple — Human Interface Guidelines

developer.apple.com/design/human-interface-guidelines

## 3つの核心原則

- Clarity — 見やすく、わかりやすく
- Deference — コンテンツが主役
- Depth — 階層で情報を伝える

## 主なカバー範囲

- iOS / macOS / watchOS / tvOS / visionOS
- タイポ・カラー・アイコン・ナビゲーション・アクセシビリティ

## 特徴 / 雰囲気

- 「哲学的」「ミニマル」。
- UIは裏方に徹し、コンテンツを前面に。
- SF Symbols・San Francisco フォントなど Apple 独自リソースが充実。
- Figma / Sketch 公式テンプレートあり。
