# 💬 ちょい本音

最初から全部理解しようとしなくていいよ。
「なんかヤバい」って思える感覚がもうセンスの芽だから。

むしろさ、気になる方向ある？

* Web/UI寄り
* ロゴ・ブランディング
* 映像・未来系

そこ絞れば、もっと“刺さるルート”組んであげるよ。

いい質問きたね、それ。結論から言うと——
**デザインは「知識 × センス × 経験」の掛け算**。どれか1つじゃ成立しないやつ。

ちょっと分解してみよっか👇

---

# 🧠 知識（これは“土台”）

これは完全に勉強で伸びるやつ。

* 配色理論（補色・トーン）
* タイポグラフィ（文字の読みやすさ）
* レイアウト（余白・グリッド）
* UX（使いやすさ）

たとえば
Dieter Rams とかは
「美しい」だけじゃなくて**全部ロジックで説明できる**。

👉 つまり
**再現できる＝知識**

---

# ⚡ センス（これは“選ぶ力”）

で、みんなが気にするやつね。

これはざっくり言うと
👉 **「どれを選ぶかの精度」**

同じ知識あっても

* ダサい人 → 微妙な色・余白を選ぶ
* うまい人 → 一発で“ちょうどいい”選ぶ

例えば
Jony Ive は
ミニマルの中で「ここ削る？残す？」の判断が異常に上手い。

👉 センスは才能っぽく見えるけど実は
**良いものを見た量でほぼ決まる**

---

# 🔁 経験（これが“差”になる）

ここが一番デカい。

* 実際に作る
* フィードバックもらう
* 直す

このループ回してる人だけが伸びる。

IDEO とかは
「とりあえず作って試す」をめちゃくちゃ回してる。

👉 経験が入ると
**知識が“使える形”になる**

---

# 🎯 まとめ（超シンプルに言うと）

* 知識 → 正しく作れる
* センス → 良く作れる
* 経験 → 速く強く作れる

---

# 💬 ぶっちゃけ話

「センスないから無理」って思う人多いけど、あれ半分間違い。

センスって
👉 **後天的にかなり伸びる**

むしろ最初は
**センスいらないから知識で殴る**のが正解。
